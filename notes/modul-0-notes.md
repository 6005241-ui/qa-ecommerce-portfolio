## Разбор команд Git

### Настройка Git (первый раз при установке)

```bash
git config --global user.name "Твоё Имя"
```
Задаёт **ваше имя** для всех репозиториев. Git будет подписывать ваши коммиты этим именем.

```bash
git config --global user.email "твоя.почта@example.com"
```
Задаёт **вашу почту** для всех репозиториев. Нужна для идентификации автора коммитов.

---

### Работа с файлами

```bash
git add .
```
**Добавляет все изменённые файлы** в индекс (подготовку к коммиту). Точка `.` означает "все файлы в текущей папке".

---

### Сохранение изменений

```bash
git commit -m "Initial setup: project structure and README"
```
**Создаёт коммит** (сохранение) с сообщением.

**Флаг `-m`** означает **"message"** (сообщение). Позволяет сразу написать описание коммита в кавычках. Без этого флага Git откроет редактор для ввода сообщения.

---

### Отправка на GitHub

```bash
git push
```
**Отправляет ваши коммиты** на удалённый репозиторий (GitHub).

---

### Работа с ветками

```bash
git checkout -b module-0-setup
```
**Создаёт новую ветку** `module-0-setup` и **переключается** на неё.

**Флаг `-b`** означает **"branch"** (ветка). Без него команда только переключается на существующую ветку. С флагом `-b` — создаёт новую и сразу переходит на неё.

---

### Отправка новой ветки

```bash
git push --set-upstream origin module-0-setup
```
**Отправляет ветку** `module-0-setup` на GitHub и **связывает** локальную ветку с удалённой.

После этого можно просто писать `git push` без указания ветки.

---

### Переключение между ветками

```bash
git checkout main
```
**Переключается** на ветку `main`.

---

### Получение изменений с GitHub

```bash
git pull
```
**Скачивает изменения** с GitHub и **объединяет** их с вашей локальной веткой.

---

## Краткая схема работы

1. **Настройка** → `git config` (один раз)
2. **Создать ветку** → `git checkout -b имя-ветки`
3. **Изменить файлы** → работаем в редакторе
4. **Добавить в индекс** → `git add .`
5. **Сохранить** → `git commit -m "сообщение"`
6. **Отправить** → `git push --set-upstream origin имя-ветки`
7. **Вернуться в main** → `git checkout main`
8. **Получить обновления** → `git pull`
